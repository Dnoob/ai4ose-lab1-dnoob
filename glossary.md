# 术语表

| 术语 | 解释 |
|------|------|
| **Rust** | 一门编程语言，本实验用它写操作系统内核 |
| **cargo** | Rust 的项目管理工具。`cargo build` 编译，`cargo run` 运行 |
| **crate** | Rust 的第三方库，cargo 会自动从网上下载安装 |
| **Cargo.toml** | 项目配置文件，声明项目名称、依赖哪些 crate |
| **QEMU** | 硬件模拟器，能在你的电脑上模拟出一台 RISC-V 计算机。我们的内核就跑在这台"虚拟电脑"里 |
| **RISC-V** | 一种 CPU 架构（类似 x86、ARM），本实验的目标硬件 |
| **SBI** | 运行在最底层（M-mode）的固件，为操作系统提供基础硬件服务（输出字符、关机等） |
| **M-mode** | Machine Mode，RISC-V 最高权限等级，SBI 固件在这里运行 |
| **S-mode** | Supervisor Mode，操作系统内核在这里运行 |
| **U-mode** | User Mode，最低权限等级，用户程序在这里运行 |
| **ecall** | RISC-V 指令，用于向更高权限等级发起请求 |
| **裸机** | 没有操作系统的硬件环境，程序直接跑在硬件上 |
| **交叉编译** | 在一种平台（如 x86 电脑）上编译出另一种平台（如 RISC-V）能运行的程序 |
| **链接脚本** | 告诉编译器把代码放在内存的哪个位置 |
| **`#![no_std]`** | Rust 属性，表示不使用标准库（因为裸机没有操作系统支撑标准库） |
| **`#![no_main]`** | Rust 属性，表示不使用标准 main 入口（因为裸机没有 C runtime） |
